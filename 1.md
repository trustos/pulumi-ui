Thought for 48s

For a fully Svelte-compatible, very customizable node/graph editor, the best choice right now is Svelte Flow. It is the most convincing option on both popularity and maintenance. Its maintainers position it as a customizable Svelte component for node-based editors, and the project is part of the larger xyflow ecosystem, which has 35.8k GitHub stars and 12.2k GitHub dependents. The Svelte-specific site shows Svelte Flow 1.5.0 with about 70k weekly installs, and its v1 line was rebuilt for Svelte 5.

Why Svelte Flow stands out:
It is explicitly built for node-based editors and interactive diagrams, ships with the common things you usually need out of the box, and is marketed as “infinitely customizable.” The repo also shows a mature package structure with dedicated Svelte support in the mono-repo, plus active releases in 2026 via @xyflow/system. That combination usually matters more in real projects than “feature bullets” alone, because it means better docs, more examples, and a healthier long-term ecosystem.

My ranking for your use case:

1. Svelte Flow — best overall
   Best when you want a polished canvas editor in Svelte with strong ecosystem momentum, modern Svelte support, and lots of room for custom nodes, edges, controls, and behaviors. It also benefits from feature parity efforts with React Flow, which is a major advantage because a lot of patterns and community knowledge transfer over.

2. Rete.js + rete-svelte-plugin — best for “true node editor engine” flexibility
   Rete is more of a framework for visual programming than just a diagram component. It supports Svelte through rete-svelte-plugin, and its docs emphasize plugin architecture, dataflow/control-flow processing, validation, history, minimap, comments, reroute, and more. If your product is closer to a workflow builder / visual programming environment / rule engine / automation editor, Rete can be stronger than Svelte Flow conceptually. The tradeoff is complexity: it is more “build your system” than “drop in a canvas component.”

3. Svelvet — best lightweight native-Svelte alternative, but weaker confidence
   Svelvet is genuinely Svelte-native and very customizable, with 2.8k GitHub stars and docs that emphasize custom nodes, anchors, edges, Mermaid-based flowcharts, and flexible composition with normal Svelte syntax. But I would be careful choosing it for a serious long-term product today: npm/search signals show activity, yet GitHub releases look stale, and there are open compatibility/bug concerns around newer Svelte versions and canvas behavior. That makes it feel more risky than Svelte Flow for a production-heavy app.

So the practical answer is:

Most popular + safest pick: Svelte Flow
Most customizable at the architecture/engine level: Rete.js with Svelte plugin
Interesting native-Svelte alternative: Svelvet, but I would not rank it first today for a serious product.

My recommendation based on what you asked:
If you want a customizable app UI graph editor in Svelte, pick Svelte Flow.
If you want a full visual-programming/workflow engine and are okay with more setup, evaluate Rete.js.
I would only choose Svelvet if you strongly prefer its API style and are comfortable accepting more maintenance risk.

A simple decision rule:

App builder / workflow canvas / drag-drop editor UI → Svelte Flow
Node execution engine / dataflow / rule system / compiler-like graph → Rete.js
Small native-Svelte prototype → Svelvet

For your Pulumi/Nomad YAML example, I would split the question into two parts:

Best for visualizing and editing this graph in Svelte

Svelte Flow

Why:
Your structure is fundamentally a resource dependency graph with:

nodes = resources / config groups / variables / outputs
edges = references like ${nomad-vcn.id}, ${nomad-compartment.id}, dependsOn, implicit property links
side panels = settings/properties/meta/config
grouping = IAM / Network / NSGs / Compute / Volumes / NLB
generated nodes = range, if, templated expansions
custom rendering = different resource cards per OCI type

That is exactly the sort of thing Svelte Flow is very good at: interactive node-based UIs, custom nodes, custom edges, nested graphs/subflows, minimap/controls, and Svelte-native component rendering.

Best if you want the graph to have “engine semantics”

Rete.js

Why:
If your editor is not just a canvas, but a schema-aware programmable graph system where the graph itself enforces:

allowed connection rules
typed ports
validation rules
dependency semantics
dataflow/control-flow style processing
graph execution / compilation pipeline

then Rete is stronger architecturally. Its own docs frame it as a framework for visual programming, with processing based on dataflow and control flow, and its Svelte plugin supports modern Svelte.
